## 1. 项目概述

### 1.1 什么是太虚幻境

**太虚幻境**是一款基于大语言模型（LLM）驱动的虚拟世界人生模拟器。它通过先进的 AI 技术，构建了一个可以无限推演的文字虚拟世界。玩家可以在任意类型的世界中体验完整的人生经历——从历史穿越到奇幻冒险，从武侠江湖到修仙问道，从现代都市到末日生存，一切皆由玩家定义。

与传统的文字冒险游戏不同，太虚幻境中的世界是**活的**：NPC 拥有自主意识，世界会自行演化，每一个选择都可能引发蝴蝶效应。

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| 🌍 **无限世界生成** | 支持 8 种预设世界类型 + 自定义世界，AI 自动生成完整世界观 |
| 🧠 **自主 NPC 系统** | 每个 NPC 都有独立人格、记忆、目标，会自主行动和演化 |
| 📖 **动态叙事引擎** | 多智能体协作叙事，叙事审查，伏笔生命周期，连续性审计 |
| 🔄 **闭环学习系统** | 系统从叙事中学习，持续优化叙事质量和玩家体验 |
| 🦋 **蝴蝶效应** | 玩家的每个选择都可能引发连锁反应，高影响行为设有审批门 |
| 🧩 **插件化架构** | 战斗系统、经济系统、天气系统等可插拔扩展 |
| 🎨 **多风格叙事风格 | 章回体、网文爽文、严肃文学等 6 种叙事风格 |
| 🗺️ **世界地图与图谱** | 可视化世界地图、NPC 关系图谱、名人谱 |

## 2. 核心理念

### 2.1 设计哲学

太虚幻境的设计理念源于对"真实虚拟世界"的追求——我们相信，一个真正的虚拟世界应该是：

- **有机的：世界有自己的运行规律，不会因为玩家的存在而存在，也不会因为玩家的离开而停止
- **自主的：NPC 有自己的思想、记忆和目标，他们会基于自己的性格做出决策
- **连贯的**：世界有记忆，历史会留下痕迹，行为会产生后果
- **可塑的**：玩家的选择能够真正改变世界，而不是走预设的剧情分支
- **无限的**：没有固定的剧情线，玩家可以做任何想做的事

### 2.2 技术愿景

我们的愿景是打造一个**真正的数字平行宇宙**，其中：
- 每个人都可以拥有自己的虚拟人生
- 每个世界都有独特的历史和未来
- 每次体验都是独一无二的
- AI 不只是讲故事的工具，更是世界的运行法则

## 3. 系统架构

### 3.1 整体架构

太虚幻境采用**分层模块化架构**，主要分为以下几层：

```
┌─────────────────────────────────────────────────┐
│                 前端交互层                        │
│  (HTML/JS + Alpine.js + WebSocket)          │
├─────────────────────────────────────────────────┤
│                 API 路由层                       │
│  (FastAPI + REST API + WebSocket)          │
├─────────────────────────────────────────────────┤
│                 游戏引擎层                      │
│  (GameEngine + 事件总线 + 服务注册中心)      │
├─────────────────────────────────────────────────┤
│              核心业务模块层                     │
│  世界生成 │ 叙事引擎 │ NPC 系统 │ 记忆系统   │
│  玩家系统 │ 任务系统 │ 经济系统 │ 战斗系统   │
├─────────────────────────────────────────────────┤
│              基础服务层                           │
│  LLM 路由 │ 向量数据库 │ 关系数据库 │ 缓存    │
└─────────────────────────────────────────────────┘
```

### 3.2 核心设计模式

#### 3.2.1 Mixin 模式

GameEngine 采用 Mixin 模式，将不同功能拆分到各个 Mixin 中：

- `SaveMixin` — 存档相关功能
- `WorldGenMixin` — 世界生成功能
- `CharacterCardMixin` — 角色卡功能
- `SubsystemQueryMixin` — 子系统查询功能

#### 3.2.2 事件总线

`EventBus` 提供子系统间的发布/订阅通信，实现模块间解耦。

#### 3.2.3 服务注册中心

`ServiceRegistry` 统一管理所有服务的生命周期和依赖关系。

#### 3.2.4 插件系统

基于钩子（Hook）机制的插件系统，支持功能的即插即用。

## 4. 核心技术模块

### 4.1 游戏引擎（GameEngine）

GameEngine 是整个系统的协调层，负责：
- 游戏状态管理
- 子系统协调
- 回合处理
- 存档管理
- 事件分发

### 4.2 回合处理器（TurnProcessorV2）

回合处理器是游戏的核心流程引擎，采用流水线（Pipeline）模式处理每一个玩家回合：

```
玩家输入 → 意图识别 → 行动校验 → 叙事生成 → 状态更新 → 世界演化 → NPC 结果输出
```

### 4.3 世界管理器（WorldManager）

负责世界的整体演化：
- 时间推进
- 世界事件生成
- NPC 离线演化
- 经济系统波动
- 天气系统变化

## 5. AI 智能体系统

### 5.1 多智能体架构

太虚幻境采用多智能体（Multi-Agent）架构，每个角色都是独立的智能体：

| 智能体 | 职责 |
|--------|------|
| 🌍 **WorldAgent** | 世界级智能体，负责世界演化、宏观事件生成 |
| 🎭 **NarrativeAgent** | 叙事智能体，负责剧情推进和场景描写 |
| 👤 **PlayerAgent** | 玩家代理，处理玩家输入和状态管理 |
| 🤖 **NPCAgent** | NPC 智能体，每个 NPC 独立决策和行动 |

### 5.2 NPC 智能体系统

#### 5.2.1 NPC 性格系统

每个 NPC 都拥有完整的人格模型：

- **MBTI 性格类型**：16 种人格类型，影响决策风格
- **性格描述**：详细的性格特质描述
- **说话风格**：独特的语言表达方式
- **对话示例**：AI 学习的对话样本
- **价值观与目标**：长期目标和行为动机
- **技能与能力**：拥有的技能和能力等级

#### 5.2.2 NPC 决策流程

```
感知环境 → 检索记忆 → 反思洞察 → 技能匹配 → 目标评估 → 行动规划 → 执行行动
```

#### 5.2.3 NPC 自主演化

NPC 即使在玩家不交互时也会：
- 自主移动和探索
- 与其他 NPC 互动
- 学习新技能
- 建立和改变关系
- 经历生老病死

### 5.3 多智能体叙事

在关键剧情时，系统会调用 3 个智能体协作生成叙事：

- **叙事者**：主导叙事视角
- **角色一致性审查者**：确保角色行为符合人设
- **剧情逻辑审查者**：确保剧情逻辑自洽

## 6. 记忆与学习系统

### 6.1 分层记忆系统

借鉴认知科学的记忆模型，系统采用三层记忆架构：

| 记忆层级 | 存储方式 | 特点 | 半衰期 |
|----------|----------|------|--------|
| **工作记忆** | 内存 | 高重要性核心记忆，每次 prompt 注入 | - |
| **情景记忆** | ChromaDB 向量库 | 具体事件记忆，带重要性评分 | 30 天 |
| **语义记忆** | SQLite | 身份/关系/世界观等长期稳定知识 | 长期 |

### 6.2 记忆评分公式

记忆检索采用综合评分算法：

```
score = 向量相似度 × 0.45 + 重要性 × 0.25 + 时间衰减 × 0.15 + 情感权重 × 0.10 + 访问频率 × 0.05
```

### 6.3 NPC 程序性记忆

NPC 从经验中学习，形成程序性记忆：

```
记录"动作-上下文-结果"三元组
    ↓
相似记忆自动合并（不膨胀）
    ↓
BranchPlanner 规划时参考历史经验
    ↓
成功率高的动作优先选择
```

### 6.4 记忆整理器（Memory Curator）

定期整理冗余/过时记忆：
- 提取行为模式
- 合并相似记忆
- 归档过时记忆
- 保持记忆库健康

### 6.5 闭环学习系统

系统具备自我改进能力：

```
叙事生成 → 回顾分析 → 教训提取 → 注入 prompt → 更好的叙事
    ↑                                              |
    └──────────────── 闭环 ─────────────────────────┘
```

- 每 N 回合自动回顾最近叙事
- 检查一致性、角色深度、节奏感
- 提取玩家偏好和叙事质量教训
- 教训直接注入到下一次叙事生成

## 7. 世界生成引擎

### 7.1 世界类型

支持 8 种预设世界类型：

| 世界类型 | 说明 | 经济重心 |
|----------|------|----------|
| 🏯 **历史穿越** | 穿越到真实历史朝代，体验古代生活 | 农业、商业、手工业 |
| 🧙 **奇幻冒险** | 魔法、龙、精灵、矮人的奇幻世界 | 魔法材料、神器、冒险公会 |
| 🚀 **科幻未来** | 太空探索、赛博朋克、AI社会 | 芯片、义体、数据 |
| 💀 **末日生存** | 文明崩塌后的生存挣扎 | 弹药、食物、净水、燃料 |
| ⚔️ **武侠江湖** | 刀光剑影、恩怨情仇的江湖世界 | 镖局、酒楼、武林秘籍 |
| 🏔️ **修仙问道** | 炼气、筑基、金丹、元婴的修真世界 | 灵石、丹药、法宝 |
| 🏙️ **都市生活** | 现代都市的日常与冒险 | 金钱、人脉、资源 |
| 🌟 **都市异能** | 现代都市中的超能力者世界 | 异能资源、秘密组织 |
| ✨ **自定义世界** | 由玩家自由描述的任意世界 | 由玩家定义 |

### 7.2 世界生成内容

AI 生成完整的世界设定包括：
- 世界名称与概述
- 地理环境与气候
- 历史背景与当前局势
- 势力分布与派系
- 经济体系
- 文化风俗
- 重要地点与地图
- 初始 NPC 阵容
- 世界规则与力量体系

### 7.3 世界任务板

世界事件自动生成任务，NPC 自动认领执行：

```
叛军入侵事件
  → 任务"指挥防御"（需求：武将）  → 武将NPC认领
  → 任务"筹集军需"（需求：商人）  → 商人NPC认领
  → 任务"安抚民心"（需求：官员）  → 官员NPC认领
```

## 8. 叙事系统

### 8.1 叙事风格

支持 6 种预设叙事风格 + 自定义风格：

| 风格 | 特点 |
|------|------|
| **章回体** | 半文半白，《三言二拍》风格 |
| **半古半文** | 文言句式+白话叙事 |
| **大白话** | 现代口语，轻松幽默 |
| **严肃文学** | 冷峻克制，余华/莫言风格 |
| **网文爽文** | 快节奏，爽点密集 |
| **诗化散文** | 意境优先，诗意优美 |
| **自定义** | 玩家自由定义风格描述 |

### 8.2 叙事视角

支持三种叙事视角：
- 第二人称 — "你"（沉浸感强，推荐）
- 第三人称 — 角色名
- 第一人称 — "我"

### 8.3 叙事审查器（Narrative Reviewer）

每 N 回合自动审查叙事质量：
- 检查角色一致性
- 检查剧情连贯性
- 检查叙事节奏感
- 提取玩家偏好
- 生成改进建议

### 8.4 伏笔生命周期

系统自动管理伏笔的生命周期：
- 伏笔埋设
- 伏笔酝酿
- 伏笔提醒（微妙/正常/明显三种模式）
- 伏笔回收
- 陈旧伏笔处理

### 8.5 连续性审计

定期审计世界和角色的连续性，确保：
- 时间线一致
- 角色行为一致
- 世界规则一致
- 因果关系合理

## 10. 技术栈与部署

### 10.1 技术栈

**后端：**
- Python 3.11+
- FastAPI — Web 框架
- Uvicorn — ASGI 服务器
- Pydantic — 数据验证
- Aiohttp — 异步 HTTP 客户端

**数据库：**
- SQLite — 元数据存储
- ChromaDB — 向量数据库（记忆检索）

**AI 模型：**
- 大语言模型（LLM）— 叙事生成、NPC 决策、世界生成
- 嵌入模型（Embedding）— 文本向量化、记忆检索
- 文生图模型（可选）— 场景插图生成

**前端：**
- HTML5 + CSS3 + JavaScript
- Alpine.js — 轻量级响应式框架
- Cytoscape.js — 关系图谱可视化
- WebSocket — 流式输出

### 10.2 LLM 路由系统

智能 LLM 路由系统：
- **主力模型**：复杂任务（叙事生成、世界生成）
- **廉价模型**：简单任务（分类、评分）
- **对话模型**：游戏内叙事/NPC对话/选项生成
- 自动降级：主模型失败时自动切换到备用模型
- 配置管理：支持多配置文件切换

### 10.3 部署方式

**本地部署：
```bash
pip install -r requirements.txt
python server.py
```

访问地址：http://localhost:8004

**一键启动：**
- Windows: 双击 `start.bat`
